Preguntas y respuestas del Laboratorio 6

  1. ¿Por qué es una buena práctica usar JavaScript para chequear que sean válidos los inputs de las formas antes de enviar los datos al servidor?
    Es buena practica porque mejora la experiencia del usuario al proporcionar retroalimentación inmediata sobre posibles errores en los datos ingresados. Reduce la carga en el servidor al evitar enviar datos no válidos y Puede ayudar a prevenir ataques de seguridad al filtrar datos maliciosos antes de enviarlos al servidor
  2. ¿Cómo puedes saltarte la seguridad de validaciones hechas con JavaScript?
    3. Puedes desactivar JavaScript en tu navegador, modificar el código JavaScript del cliente para evitar o eludir las validaciones. Además, puedes enviar datos directamente al servidor sin utilizar el formulario HTML o enviar datos directamente al servidor sin utilizar el formulario HTML
  3. Si te puedes saltar la seguridad de las validaciones de JavaScript, entonces ¿por qué la primera pregunta dice que es una buena práctica?
    4. Porque aun así proporcionan una primera línea de defensa contra errores de entrada comunes. Además, ayudan a mejorar la experiencia del usuario al proporcionar retroalimentación inmediata, y aunque, no pueden evitar todos los ataques, pueden disuadir a los atacantes menos sofisticados y reducir el riesgo de errores accidentales.
